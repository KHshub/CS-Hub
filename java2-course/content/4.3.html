4.3 - The Difference Between Abstract and Interface


There are many differences between Java's interfaces and abstract classes, they may have some similarities but they are not the same.

1. In the case of extends

  • If you have a class that inherits from a class, the subclass is free to Override the functions it inherited.
  • If you have a class that inherits from the class of its type abstract, the Subclass is forced to Override for functions defined as abstract.
  • If you have an Interface that inherits from an Interface, the sub-interface cannot override the functions it inherited.

Example:

interface A { } interface B { } interface C extends A, B { }

1. In the case of implements

  • If you have a class running Interface, the subclass is forced to Override the functions it inherited.
  • If you have an abstract class that implements an interface, the subclass is free to Override the functions it inherited.

Example:

interface A { } interface B { } class C implements A, B { }

The Differences Between Interfaces and Abstract Classes:

Interface Abstract Class
Implicitly abstract and cannot have implimentations Can have instance methods that implement a default behavior
Variables declared in a Java interface are by default final May contain non-final variables
Members of a Java interface are public by default Can have the usual favors of a class members like private, protected, etc.
Should be implemented using keywork "implements" Should be extended using keyword "extends"
Can extend another Java interface only Can extend another Java class and implement multiple
Absolutely abstract and can't be instantiated Can't be instantiated but can be invoked if a main() exists
Slow as it requires extra indirection Comparatively fast
