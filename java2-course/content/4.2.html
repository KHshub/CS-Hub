4.2 - Interfaces


A Java Interface contains static constants and abstract methods.

its syntax is:

Interface myInterface { }

Why do we use interfaces?

When we say an interface has abstract methods, it means the method has no body. So, in interfaces, the methods are only declared (have the method name and signature), we never write the body. This might make you think, why do we use interfaces then? Well, there are two main reasons:

  1. Interfaces are one of the ways we achieve abstraction in Java
  2. Interfaces are used to achieve pure or complete abstraction, multiple inheritance, and polymorphism

Interface Properties:

  • Will have only method declaration(abstract methods) and constant attributes in it.
  • All the variables inside Interface are public, static, and final even if we don’t specify anything.
  • Even when we don't provide any specifications, all methods are public and abstract.
  • Can’t change these default access modifiers
  • Every method inside an interface is marked as abstract only

Example:

Interface myInterface { int X = 1; void print(); }
All of the interface's methods must be overridden in the implementing class because they are all abstract.. ( But when do we have to do Override and when do we not?)
