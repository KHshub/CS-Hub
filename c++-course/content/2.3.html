2.3-Arithmetic expressions


Arithmetic Operations

C++ supports addition, subtraction, multiplication, division, and modulus division.

  • In a single arithmetic expression, various data types may be employed.
  • Arithmetic operators are binary operators


    • Operators Example: +,-,*,/,%,=,etc..

    Operands Example: var1,var2,var3,etc..
    • Binary operators: Require two operands.
    • Unary operator: Requires only one operand.
    • Negation operator (-): Reverses the number's sign.

    Expression Types

  • Expression: Any set of operators and operands that can provide a value upon evaluation.
    • Expression Example:

    #include <iostream>
    using namespace std;
    int main()
    {

    //varible to store expression(result)

    result = a + b / c

    }

  • The expression is given a name based on the data type employed (integer expression,...) if all of the operands are of the same type.
  • Mixed-mode expression: Contains floating-point and integer operands, producing a double-precision value.
  • Integer division: Produces an integer result, discarding all fractional remainders (truncated)
    • Example: The result is 1.2 but it is integer so it comes back 1.
  • Modulus (remainder) operator: Returns only the remainder.

    • Operator Precedence & Associativity

  • Rules for writing arithmetic expressions:
    • Never place two consecutive (one after another) binary arithmetic operators side by side.
    • Use parentheses to form groupings (Contents within parentheses are evaluated first), ( a * b ) .
    • Parentheses may be nested inside of other parentheses (Evaluated from innermost to outermost).
    • Use the * operator for multiplication, not parentheses.
  • The precedence of operators is used to evaluate expressions with multiple operators:
    • All negations occur first.
    • Then multiplication, division, and modulus, left to right.
    • Addition and subtraction are last, left to right.
  • Associativity: The sequence in which operators with similar precedence are assessed.

    • Assignment Operations

    Structure: variable = Expression
    Example:   count      =   var + 1

    Rules:
    • operator: Equal (=) sign
    • If the same variable is utilized, the new value will take the place of the old one.
    • The expression can be any expression that can be evaluated to a value.
    • Terminated with a semicolon.
    • Multiple assignments in the same expression are possible.
    Accumulation statement:   Has accumulating effect,
    Example: sum = sum + new_value

    Short cuts to assignment operator :

    1. ( x += 1 ) identical to x= x + 1
    2. ( x -+ 1 ) identical to x= x - 1
    3. ( x *= 1 ) identical to x= x * 1
    4. ( x /= 1 ) identical to x= x / 1
    5. ( x %= 1 ) identical to x= x % 1

      6. And many more
    Counting statement:   Adds a fixed value to the variable’s current value,
    Example: count = count + 1
    • Increment operator (++) : Unary operator, when the variable is increased by one.
      • Prefix increment operator: appears before the variable ( ++i )
      • Postfix increment operator: appears after the variable ( i++ )
    • Decrement operator (--) : Unary operator ,when the variable is decreased by one.
      • Prefix decrement operator: appears before the variable ( --i )
      • Postfix decrement operator: appears after the variable ( i-- )
