4.2-Repetition


Basic Loop Structures

  • Loop : A portion of code that repeats and whose repetitions are limited by a conditional test.
  • Repetition structure has four required elements:
  1. Repetition statement include: while , for & do while
  2. Condition to be evaluated , can be tested :
    • Pretest (entrance-controlled):Condition is tested first; if false, statements in the loop body are never executed (while & for loops)
    • Posttest loop: Condition is tested after the loop body statements are executed; loop body always executes at least once (do while loop)
  3. Initial value for the condition
    • Fixed-count loop: Loop processing is done for a set number of times.
    • Variable-condition loop: Dependent on the value of a variable, the number of repeats.
  4. Loop termination, To prevent an infinite loop, something in the body of the loop must force the condition to change.
    • break statement:makes switch, while, for, and do-while statements immediately break or exit.
      • Despite breaching the rules of pure structured programming, it is nonetheless helpful for getting out of loops when an uncommon circumstance is found.
    • continue statement: When encountered inside a loop, control jumps to the beginning of the loop for next iteration, skipping the execution of statements inside the body of loop for the current iteration.

For statement:

  • A loop that responds to condition changes and has a fixed count condition.
  • Syntax

    • for (initializing list(i=0); expression(i<10); altering list(i++)) {
    statement;
    }

  • Initializing list: Sets a counter's starting value.
    The counter starting value must be provided before entering the for loop if initializing list is lacking.
  • Expression: Contains the max or min value the counter can have; determines when the loop is finishes Omitting the expression will result in an infinite loop
  • Altering list: Provides the increment value that is added or subtracted from the counter in each iteration If altering list is missing, the counter must be altered in the loop body

    • Techniques are suitable for pretest loops (for & while):

  • Interactive input within a loop: The while or for loop contains a cin statement.
  • Selection within a loop: using a for or while loop to iteratively cycle through a set of numbers and choose those that satisfy certain requirements.

    • Nested loop

  • A loop contained within another loop.
  • No statements from the inner loop may overlap with any assertions from the outer loop.
  • Different variables must be used to control each loop.
  • The inner loop cycles through all of its iterations for each iteration of the outer loop.

    • Do while loop

  • A post-test loop runs each statement at least once.
  • The condition is assessed at the conclusion of the loop.
  • Syntax:

    • do {
    statement; }
    while (expression);


  • A post-test loop runs each statement at least once.


    • Validity Checks

  • Filtering user input and performing data validation tests are both useful.
  • Can be improved by an if-else clause.
