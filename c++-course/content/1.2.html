1.2-Software Development Steps


Definitions:


Software development procedure: Help developers understand problem to be solved and create an effective , appropriate software solution.

Software engineering: Creates programs and systems that are understandable, effective, dependable, and maintainable - Uses software development methodology to accomplish its goals.



First you need?

Program requirement: Request for a program or a statement of a problem.


Then you start with Development and Design Phase One

Phase one consists of

  • Analysis
    1. Determine and comprehend the output items the program needs to deliver.
    2. Determine the input items.
    3. Both components are referred to as the input/output (I/O) problems.

  • Design
    1. Select the exact set of steps, "algorithm,” or a “flowchart” to solve the problem.
    2. Refine the algorithm , Start with the initial solution in the analysis step until you have an acceptable and complete solution.
    3. Check the solution.

  • Coding
    • Which means translating the design into a C++ program or any language Statements Types
    • Sequence: Establishes the sequence in which instructions are carried out.
    • Selection: Enables the choice of a different procedure depending on a condition.
    • Iteration (looping): Allows the same operation to be repeated based on some condition.
    • Invocation: Involves calling a series of statements as necessary.

  • Testing
    • How to check that something is correct and that the requirements are met

    “ Takes The Most Effort ”

    Bug: A program error.

Debugging: The procedure for identifying a problem, fixing it, and then confirming the fix.

    Documentation Phase Two Main documents for every problem solution:

  • Program description
  • Algorithm development and changes
  • Well-commented program listing
  • Sample test runs
  • Users' manual

    Maintenance Phase Three Ongoing correction of newly discovered bugs,Revisions to meet changing user needs, Addition of new features.

  • Longest phase
  • Maybe the primary source of revenue
  • Good documentation vital for effective maintenance

Backup: Process of routinely making copies of computer code and documenting it.

Backup copies: insurance against loss or damage ; better to use off-site storage for additional protection.
