4.3-Repetition

Basic Loop Structures

  • Loop : A section of repeating code, whose repetitions are controlled by testing a condition
  • Repetition structure has four required elements:
    1. Repetition statement include: while , for & do while
    2. Condition to be evaluated , can be tested :
      • Pretest (entrance-controlled):Condition is tested first; if false, statements in the loop body are never executed (while & for loops)
      • Posttest loop: Condition is tested after the loop body statements are executed; loop body always executes at least once (do while loop)
    3. Initial value for the condition
      • Fixed-count loop: Loop is processed for a fixed number of repetitions
      • Variable-condition loop: Number of repetitions depends on the value of a variable
    4. Loop termination , Something in loop body must cause the condition to change, to avoid an infinite loop
      • break statement:Forces an immediate break, or exit, from switch, while, for, and do-while statements
        • Violates pure structured programming, but is useful for breaking out of loops when an unusual condition is detected
      • continue statement: When encountered inside a loop, control jumps to the beginning of the loop for next iteration, skipping the execution of statements inside the body of loop for the current iteration.

    For statement:

  • A loop with a fixed count condition that handles alteration of the condition
  • Syntax

    • for (initializing list; expression; altering list) {
    statement;
    }

  • Initializing list: Sets the starting value of a counter
    If initializing list is missing, the counter initial value must be provided prior to entering the for loop
  • Expression: Contains the max or min value the counter can have; determines when the loop is finishes Omitting the expression will result in an infinite loop
  • Altering list: Provides the increment value that is added or subtracted from the counter in each iteration If altering list is missing, the counter must be altered in the loop body

    • Techniques are suitable for pretest loops (for & while):

  • Interactive input within a loop: Includes a cin statement within a while or for loop
  • Selection within a loop: Using a for or while loop to cycle through a set of values to select those values that meet some criteria

    • Nested loop

  • A loop contained within another loop
  • All statements of the inner loop must be completely contained within the outer loop; no overlap allowed
  • Different variables must be used to control each loop
  • For each single iteration of the outer loop, the inner loop runs through all of its iterations

    • Do while loop

  • Is a posttest loop that continues while the condition is true
  • Condition is tested at the end of the loop
  • Syntax:

    • do {
    statement; }
    while (expression);


  • All statements are executed at least once in a posttest loop

    • Validity Checks

  • Useful in filtering user-entered input & providing data validation checks
  • Can enhance with if-else statement
