5.1-Functions


    Interaction with a function includes:

  • Properly calling a function and passing data to it.
  • Returning values from a function after it has finished running.
  • A function is called by giving the function’s name & passing arguments in the parentheses following the function name


  • Function prototype : declaration statement for a function is
  • It can be placed with the variable declaration statements above the calling function name or in a separate header file
  • A function prototype can be used to declare functions to all calling functions.


  • calling function:
    1. kind of value that will be explicitly returned, if any
    2. Data type & order of the values the calling function should transmit to the called function
  • Requirements when calling a function include:
    1. Using the name of the function
    2. Enclosing any data passed to the function in the parentheses following the function name, using the same order & type declared in the function prototype

  • Argument : items enclosed in the parentheses of the called function
    • Arguments parameters indicated by the function header must match those passed to a function when it is called.


How a function looks like





    Returning a single value

  • Function receiving an argument passed by value :
    • Cannot accidentally change the value of the argument's variable.
    • Can process the values sent to it in any fashion & return one, & only one, “legitimate” value directly to the calling function


    Pass by value Vs Pass by reference

  • Pass by value: Signifies that you are storing a copy of the parameter's actual contents in memory, not just the parameter's actual value, which is given in.
  • Pass by reference:
    • The address of the real argument is copied and stored.

    Variable scope

  • Local variables: Variables that are created in a function but are typically only accessible to the function.
  • Scope: Program section where the identifier is "known" or valid.
  • A variable with local scope is one with storage locations set aside for it by a declaration statement inside the function that declared them
  • Storage is generated for a variable with global scope by a declaration statement outside of any function.
