4.4 - Inference Engine


Inference Engine: A set of procedures that use the representation language to infer new facts from known ones. or respond to various KB questions.


Examples of Approaches:

1. Model Checking

Model checking is a method to determine entailment. it explicitly implements the concept of entailment.

  • The algorithm is sound.
  • Complete because it works for any KB, α, and terminates (there are only a finite amount of models to examine).
  • Time complexity of the algorithm is O(2^n).
  • The space complexity is only O(n) because the enumeration is depth-first.
  • Not efficient

2. Propositional Theorem Proving (Applying logic)

Propositonal Theorem Proving applies rules of inference directly to the sentences in the KB to construct a proof without consulting models. it is a much more efficient method of determining entailment than model checking.

Two sentences α & β are logically equivalent if they are true in the same set of models, (α ≡ β )
  • e.g., P ∧ Q & Q ∧ P are logically equivalent.

If a statement is true in all models, it is valid (a tautology).
  • e.g. True, A ∨ ¬A, A => A, (A ∧ (A => B)) => B

Validity is connected to inference via the Deduction Theorem: KB ╞ α if & only if (KB ∧ α) is valid.

  • A sentence is satisfiable if it is true in some model: e.g., A ∨ B
  • A sentence is unsatisfiable if it is false in all models: e.g., A ∧ ¬A

The following pairs of sentences are equivalent and one can be replaced by the other:

  • Commutativity:
    • P ∧ Q = Q ∧ P -> Commutativity of ∧.
    • (P ∨ Q) ∨ R = P ∨ (Q ∨ R) -> Commutativity of ∨.

  • Associativity:
    • (P ∧ Q) ∧ R= P ∧ (Q ∧ R) -> Associativity of ∧.
    • (P ∨ Q) ∨ R= P ∨ (Q ∨ R) -> Associativity of ∨.

  • Identity element:
    • P ∧ True = P
    • P ∧ True = P

  • Distributive:
    • P ∧ (Q ∨ R) = (P ∧ Q) ∨ (P ∧ R) -> Distributive of ∧ over ∨.
    • P ∨ (Q ∧ R) = (P ∨ Q) ∧ (P ∨ R) -> Distributive of ∨ over ∧.

  • DE Morgan's Law:
    • ¬ (P ∧ Q) = (¬P) ∨ (¬Q)
    • ¬ (P ∨ Q) = (¬ P) ∧ (¬Q)

  • Double-negation elimination:
    • ¬ (¬P) = P



3. Searching

Searching proofs can be done instead of listing models, and it is more effective because it ignores pointless sentences.

Sound ? yes , Complete ? yes , but if the available inference rules are inadequate, then the goal is not reachable.

4. Resolution

Resolution is a single inference rule that gives a complete inference algorithm when coupled with any complete search algorithm.

  • The resolution rule applies only to clauses (disjunctions of literals).
  • Does it mean it applies to KBs & queries consisting of clauses? Every sentence of propositional logic is logically equivalent to a conjunction of clauses.
  • A sentence expressed as a conjunction of clauses is said to be in conjunctive normal form or CNF.
