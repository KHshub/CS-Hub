2.2 - Types of Agents & Environments


The first step to design agents is specifying the task environment.

PEAS -> Performance measure - Environment - Actuators - Sensors


Example: Taxi Driver

Performance Measure: Fast
Environment: Roads
Actuators: Break
Sensors: GPS


The following graph shows the types of agents & environments



Types of agents:

1. Simple reflex agent

  • Select actions based on the current percept, ignoring the rest of the percept history.
  • Simple reflex behaviors occur even in more complex environments. (connection a condition–action rule).
  • Program in schematic form, showing how condition–action rules allow agent to make the connection from percept to action.
    ( Connection a condition–action rule: The kind of connection when we go directly from current percept to an action )
  • Drawback: they can only work if decision can be made based on the current state they receive Otherwise there will be the stare of infinite loop.

2. Model-based reflex agents

  • To handle partial observability, the agent must maintain some sort of internal state that depends on the percept history
  • Updating internal state information over time requires two kinds of knowledge to be encoded in the agent program:
    1. Information about how the world evolves independently of the agent
    2. Information about how the agent’s own actions affect the world
  • This knowledge about “how the world works” is called a model of the world.
  • It is hardly possible for the agent to determine the current state of a partially observable environment exactly.

3. Goal-based agents

  • As well as current state description, agent needs some sort of goal information that describes situations that are desirable.
  • As well as current state description, agent needs some sort of goal information that describes situations that are desirable.
  • Goal-based action selection can be straightforward, but often, requires long sequence of searching.
  • Search & planning are the subfields of AI devoted to finding action sequences that achieve the agent’s goals.
  • Notice that decision making of this kind is different from the condition–action rules, as it involves consideration of the future:
    • “What will happen if I do such-and-such?”
    • “Will that make me happy?”


4. Model-based reflex agents

  • Goals alone are not enough to generate high-quality behavior in most environments.
  • A more general performance measure should allow a comparison of different states according to how happy they would make the agent.
  • Instead of “happy”, economists & computer scientists use utility.
  • In two cases, goals are inadequate but a utility-based agent can still make rational decisions:
    1. When there are conflicting goals, only some of which can be achieved
    2. When there are several goals, none of which is certain to be achieved

Types of environment:

1 Fully Observable
If an agent sensor can sense or access the complete state of an environment at each point of time
Is easy as there is no need to maintain the internal state to keep track of the history of the world.		 Partially Observable
If an agent sensor cannot sense or access the complete state of an environment at each point of time.
2 Deterministic
If an agent's current state and selected action can completely determine the next state of the environment.
Does not need to worry about uncertainty.		 Stochastic
Random in nature and cannot be determined completely by an agent
3 Epsiodic
An agent's action is divided into atomic episodes. Decisions do not depend on previous decisions/actions.
The action in each episode only depends on the episode itself.		 Sequential
An agent requires memory of past actions to determine the next best actions.
4 Static
If the environment cannot chane itself while an agent is deliberating.
Easy to deal because an agent does not need to continue looking at the world while deciding for an action.		 Dynamic
If the environment can change itself while an agent is deliberating.
Agents need to keep looking at the world at each action.
5 Discrete
If in an environment there are a limit number of percepts and actions that can be performed within it.		 Continuous
If in an environment there are an unlimited number of percepts and actions that can be performed within it.
Taxi driving is a continuous-state and continuous-time problem.
6 Single-Agent
If only one agent is involved in an environment, and operating by itself.		 Multi-Agent
If multiple agents are operating in an environment.
Can be competitive or cooperative
